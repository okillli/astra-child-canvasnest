## Migration Complexity Analysis

### What Happens to Custom CSS During Migration?

**Critical Finding:** CSS added in WordPress Customizer > Additional CSS is **theme-specific** and will be **completely lost** when switching themes.

**Your Risk:**
- 2,259 lines of custom CSS = lost
- All Phase 1-3 improvements = need rewriting
- Product grid spacing, typography, sale badges, trust signals, etc. = reset

**Preservation Options:**
1. **Plugin-based CSS:** Use "Simple Custom CSS" plugin (survives theme changes)
2. **Child theme:** Create child theme for new theme (requires rewriting all CSS)
3. **Code Snippets plugin:** For functions.php code (survives theme changes)

**Time Investment to Migrate:**
- Analyzing which CSS to keep: 4-6 hours
- Rewriting CSS for new theme: 20-30 hours
- Testing and refinement: 10-15 hours
- **Total:** 34-51 hours (approximately $1,700-2,550 at $50/hour)

## Migration Considerations (If Not Choosing Astra Pro)

### Pre-Migration Checklist:

- [ ] Backup entire site (database + files)
- [ ] Export custom CSS to text file
- [ ] Document all customizations
- [ ] Test new theme on staging site
- [ ] Plan URL structure (301 redirects if needed)
- [ ] Prepare product data for re-import
- [ ] Schedule maintenance window
- [ ] Notify customers of potential downtime

### CSS Migration Strategy:

1. **Categorize existing CSS:**
   - Theme-specific (50-60%) - needs rewriting
   - WooCommerce-specific (30-40%) - mostly transferable
   - Custom branding (10-20%) - fully transferable

2. **Rewriting approach:**
   - Start with critical pages (shop, product, cart)
   - Adapt CSS to new theme's class names
   - Use browser inspector to identify new selectors
   - Test incrementally, not all at once

3. **Tools to help:**
   - Chrome DevTools (inspect elements)
   - WP Rocket (CSS optimization)
   - Query Monitor (debug queries)
